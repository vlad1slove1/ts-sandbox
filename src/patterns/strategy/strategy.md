# Strategy (Стратегия)

## Назначение

Паттерн **Strategy** определяет семейство алгоритмов, инкапсулирует каждый из них и делает их взаимозаменяемыми. Стратегия позволяет менять алгоритмы независимо от клиентов, которые ими пользуются.

## Проблема

Когда один и тот же результат можно достичь разными способами (разными алгоритмами), и нужно иметь возможность переключаться между ними в runtime или выбирать подходящий вариант в зависимости от контекста.

## Решение

- Вынести каждый алгоритм в отдельный класс с общим интерфейсом
- Контекст (Context) хранит ссылку на стратегию и делегирует ей работу
- Клиент может подменять стратегию в любой момент

## Структура

```
┌─────────────────────┐
│     Context         │
├─────────────────────┤
│ - strategy          │
├─────────────────────┤
│ + setStrategy()     │
│ + execute()         │
└──────────┬──────────┘
           │
           │ использует
           ▼
┌─────────────────────┐
│  <<interface>>      │
│     Strategy        │
├─────────────────────┤
│ + algorithm()       │
└──────────┬──────────┘
           │
     ┌─────┴─────┬─────────────┐
     ▼           ▼             ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│StrategyA│ │StrategyB│ │StrategyC│
└─────────┘ └─────────┘ └─────────┘
```

## Преимущества

- Избегание условных операторов для выбора алгоритма
- Открыт для расширения, закрыт для модификации (Open/Closed)
- Разделение ответственности: контекст не знает деталей алгоритма
- Упрощение тестирования — стратегии можно тестировать изолированно

## Недостатки

- Увеличение числа классов
- Клиент должен понимать разницу между стратегиями

## Применимость

- Есть несколько вариантов решения одной задачи
- Нужно скрыть сложную логику выбора алгоритма
- В классе много условных ветвлений для выбора поведения
- Разные варианты алгоритма используются в разных частях приложения

## Связь с другими паттернами

- **State** — похож по структуре, но State меняет поведение объекта в зависимости от внутреннего состояния, а Strategy — по внешнему выбору
- **Decorator** — оборачивает объект, Strategy — подменяет алгоритм
- **Template Method** — наследование, Strategy — композиция
